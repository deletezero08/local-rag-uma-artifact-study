## Short manuscript description
This manuscript studies how to make local retrieval-augmented generation usable on constrained unified-memory hardware. It presents an artifact-linked systems study of backend selection, retrieval compression, context pruning, and stage-wise latency decomposition on a Mac mini M4 with 16GB unified memory.

## Suggested significance statement
The manuscript contributes a systems-oriented analysis of local RAG execution under constrained hardware conditions. Rather than introducing a single standalone retrieval method, it shows how backend choice, retrieval compression, and context pruning interact within the same execution path, and it documents both feasible and infeasible deployment routes using artifact-linked evidence.

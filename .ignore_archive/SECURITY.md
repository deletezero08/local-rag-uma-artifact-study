# Security Policy - MemoraRAG

## 核心安全理念

MemoraRAG 是一个以**隐私优先**为核心的系统。我们所有的设计都应严格遵循“本地化”原则。

## 受支持的版本

我们目前仅积极维护 `main` 分支。请确保您始终更新至最新版本以获得安全补丁。

## 漏洞定义

以下情况被视为 MemoraRAG 的严重安全漏洞：
1. **隐私泄露**: 系统在未授权的情况下将用户本地文档或提问历史发送至第三方服务器。
2. **逻辑越权**: 能够绕过 `RAG_API_KEY` 直接访问 `/api/sessions` 或读取任意文件。
3. **拒绝服务**: 构造恶意输入导致后端服务死锁或崩溃。

## 报告流程

如果您发现任何安全缺陷，请**不要**直接开设公开 Issue。这可能会让恶意用户在修复完成前利用该漏洞。

请按照以下步骤操作：
1. 编写一份简要的漏洞描述及复现 PoC。
2. 联系项目维护者（如果在 GitHub 上，请使用 Private Vulnerability Reporting 功能）。
3. 我们承诺在 48 小时内确认报告，并在 7 天内发布修复建议或正式补丁。

## 鸣谢

对于发现并负责任地披露重大安全漏洞的贡献者，我们将在 README 的致谢名单中予以标注。

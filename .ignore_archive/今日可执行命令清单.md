# 今日可执行命令清单（P0）

以下命令按顺序执行，默认在项目根目录运行：`/Users/delete/Desktop/rag_system_副本`。

## 0) 环境与快速自检

```bash
pwd
python3 --version
./.venv313/bin/python --version
./.venv313/bin/python -m py_compile scripts/check_step1_acceptance.py scripts/dual_judge_acceptance.py scripts/aggregate_results.py scripts/select_pareto.py
```

## 1) 固定本次运行ID

```bash
export RUN_ID="paper_n50_$(date +%m%d_%H%M)"
echo "$RUN_ID"
```

## 2) 执行 Step1 论文口径验收（先单裁判主流程）

说明：当前脚本参数仍是 `context_char_budget` 口径，先按现有脚本跑通，结果用于正文与图表；后续再做 token 口径重构。

```bash
HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 ./.venv313/bin/python scripts/check_step1_acceptance.py \
  --sample-size 50 \
  --iterations 3 \
  --run-id "$RUN_ID" \
  --baseline-top-n 10 \
  --baseline-budget 0 \
  --optimized-top-n 8 \
  --optimized-budget 1500 \
  --baseline-file "step1_acceptance_baseline_${RUN_ID}.json" \
  --optimized-file "step1_acceptance_optimized_${RUN_ID}.json" \
  --check-file "step1_acceptance_check_${RUN_ID}.json"
```

## 3) 执行双裁判复核（可串行，避免显存波动）

```bash
HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 ./.venv313/bin/python scripts/dual_judge_acceptance.py \
  --sample-size 50 \
  --iterations 1 \
  --optimized-top-n 8 \
  --optimized-budget 1500 \
  --judge-a-model qwen3:8b \
  --judge-b-model deepseek-r1:8b \
  --serial-judge \
  --unload-between-judges \
  --run-id "$RUN_ID" \
  --out-file "step1_dual_judge_check_${RUN_ID}.json"
```

## 4) 汇总统计与Pareto筛选

```bash
./.venv313/bin/python scripts/aggregate_results.py
./.venv313/bin/python scripts/select_pareto.py
```

## 5) 生成论文图（质量/相关性/时延）

```bash
./.venv313/bin/python scripts/plot_ablation.py
ls -lah experiments/results/figures
```

## 6) 快速检查核心结果文件是否落盘

```bash
ls -lah experiments/results | grep -E "step1_acceptance_(baseline|optimized|check)_${RUN_ID}|step1_dual_judge_check_${RUN_ID}|ablation_summary.json|pareto_selection.json"
```

## 7) 追加备案（执行后）

执行完上面命令后，至少记录以下三类结论到日志文档：

1. 本次 `RUN_ID` 与样本规模（n=50, iter=3）；  
2. 单裁判验收结论 + 双裁判复核结论；  
3. 是否满足双约束、若未满足则写明失败原因与下一步动作。

## 可选：夜间续跑（若中途被中断）

若主流程因OOM或中断，可直接复用同一 `RUN_ID` 续跑：

```bash
  --run-id "$RUN_ID" \
  --only optimized \
  --optimized-top-n 8 \
  --optimized-budget 1500 \
  --optimized-file "step1_acceptance_optimized_${RUN_ID}.json"
```

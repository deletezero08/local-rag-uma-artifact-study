## Folder Map

- `A_api/`: API usage notes and model-serving references.
- `B_deploy/`: deployment guides for Ollama, vLLM, TGI, and CUDA-related setup.
- `C_troubleshoot/`: failure modes, OOM guidance, and port issues.
- `D_design/`: architecture and design write-ups.
- `E_config/`: environment, security, quantization, and deployment config notes.
- `archive/`: frozen summaries and historical notes.
- `manuscripts/`: manuscript analysis and narrative drafts.
- `references/`: cited or referenced external materials.
- `specs/`: project-level specs and log schemas.
- `templates/`: reusable templates and source artifacts.

## How To Use It

1. Read `PROJECT_HANDOFF.md` or `PROJECT_MAP.md` first.
2. Jump to `results/README.md` if the task is evidence- or experiment-related.
3. Jump to `scripts/README.md` if the task is execution- or runner-related.
4. Prefer the currently active source of truth over archived notes when the two disagree.
